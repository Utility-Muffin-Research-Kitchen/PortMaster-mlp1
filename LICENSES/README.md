# Licenses And Provenance

The default manager package does not bundle upstream PortMaster payloads,
runtime bundles, or armhf compatibility binaries.

Generated release assets must add provenance and license notes here before they
are published.

The armhf compatibility pack generated by `make build-armhf-compat` includes a
pinned Rockchip Mali userspace blob from `tsukumijima/libmali-rockchip`. That
blob is closed-source Arm Mali driver software. The generated pack carries the
upstream Debian copyright/EULA text at:

```text
licenses/mali/libmali-rockchip-debian-copyright
```

Do not publish an armhf compatibility release asset unless that file is present
in the zip and the generated manifest records the Mali source commit, blob
SHA-256, and license SHA-256.

The MLP1 pak generated by `make package-mlp1` also includes a pinned aarch64
Rockchip Mali userspace blob from the same upstream project. It is installed to
`$USERDATA_PATH/portmaster/compat/mali/aarch64` and is used only for aarch64
Godot direct-Wayland launches, ahead of the stock rootfs Mali blob. The package
must carry the same upstream Debian copyright/EULA text at:

```text
LICENSES/mali/libmali-rockchip-debian-copyright
```
